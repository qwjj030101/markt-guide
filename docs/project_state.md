# 项目状态文档 (project_state) — 市场导购微信小程序 v3.4

## 1. 项目概述

- **项目名称**：市场导购微信小程序
- **版本**：v3.4
- **目标**：连接采购商与商户，实现供需对接的导购平台。
- **核心功能**：商铺展示、采购需求发布与响应、商户入驻与续费、商铺管理。
- **预期上线**：2026年5月底（暂定）

## 4. 风险评估与应对

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| 支付回调丢失 | 中 | 高 | 实现前端轮询 + 后台补单接口，客服可手动处理 |
| 商户过期降级定时任务未执行 | 低 | 中 | 增加监控告警，每日邮件通知执行结果 |
| 权限校验遗漏导致数据泄露 | 低 | 高 | 代码审查 + 安全测试，所有云函数强制校验 openid |
| 微信政策变化（如支付、用户信息） | 低 | 中 | 关注官方公告，及时调整 |
| 需求广场数据量过大导致加载慢 | 中 | 中 | 分页加载、索引优化、缓存热点数据 |
| Tab 页间状态共享导致数据不一致 | 低 | 中 | 使用全局变量（app.globalData）统一管理用户状态 |

## 5. 质量保证

- **代码规范**：使用 ESLint（小程序专用规则），统一命名风格。
- **代码审查**：每次合并前至少一名其他开发者审查。
- **测试覆盖**：
  - 单元测试：云函数关键逻辑（如防重、过期降级）使用 Jest。
  - 集成测试：使用微信开发者工具模拟不同角色登录。
  - 回归测试：每次迭代后执行核心功能测试用例。
