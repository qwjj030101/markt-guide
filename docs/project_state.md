# 市场导购微信小程序项目状态文档 (project_state) v4.0

## 1. 项目概述
- **项目名称**：市场导购微信小程序
- **版本**：v4.0
- **目标**：连接采购商与商户，实现供需对接的导购平台，商户免费入驻，按响应次数或包月收费。
- **核心功能**：商铺展示、需求发布与响应（含次数限制）、免费入驻、购买响应包、商铺管理。
- **当前状态**：**第五阶段（测试与上线）**

## 4. 风险评估与应对
| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| 支付回调丢失 | 中 | 高 | 前端轮询 + 后台补单接口，客服可手动处理 |
| 商户次数不足时引导购买体验不佳 | 低 | 中 | 设计清晰的弹窗文案，购买后自动返回原需求 |
| 权限校验遗漏导致数据泄露 | 低 | 高 | 代码审查 + 安全测试，所有云函数强制校验 openid |
| 微信政策变化（如支付、用户信息） | 低 | 中 | 关注官方公告，及时调整 |
| 需求广场数据量过大导致加载慢 | 中 | 中 | 分页加载、索引优化、缓存热点数据 |

## 5. 质量保证
- **代码规范**：使用 ESLint（小程序专用规则），统一命名风格。
- **代码审查**：每次合并前至少一名其他开发者审查。
- **测试覆盖**：
  - 单元测试：云函数关键逻辑（防重、扣费、过期降级）使用 Jest。
  - 集成测试：使用微信开发者工具模拟不同角色登录。
  - 回归测试：每次迭代后执行核心功能测试用例。
